🧰 Frameworks - מתי לבנות ומתי להשתמש בכלי מוכן

פריימוורק ל-AI לא מוסיף יכולת AI חדשה - הוא מוסיף שכבת הפשטה מעל קריאות API גולמיות: אורקסטרציה של שרשראות קריאות, ניהול זיכרון שיחה, אינטגרציה מובנית עם retrieval/RAG, והפשטות ל-tool calling. שלושה שמות שכדאי להכיר: LangChain - פריימוורק אורקסטרציה כללי עם דגש על "שרשראות" (chains); LlamaIndex - מתמקד בעיקר באינדוקס וניהול נתונים עבור RAG; Claude Agent SDK - ה-SDK הרשמי של Anthropic, שנותן גישה ישירה לאותם פרימיטיבים שממש בונים את Claude Code עצמו.

היתרון המרכזי: פרוטוטייפ מהיר, פחות boilerplate, אינטגרציות מוכנות. החיסרון המרכזי: שכבת ההפשטה יכולה "להסתיר" בדיוק מה שנשלח בפועל למודל, ומוסיפה תלות בקוד צד-שלישי.

היוריסטיקת החלטה: עבור סוכן אחד עם מטרה מוגדרת היטב, עבודה ישירה מול ה-API או Claude Agent SDK נותנת שליטה וברירות מלאה. עבור פרויקט עם הרבה מקורות נתונים ושרשראות מורכבות - פריימוורק כללי יכול לחסוך זמן.

🏛️ ארכיטקטורת Fullstack ל-AI

תכונת AI בפרודקשן היא לא רק "קריאה למודל" - היא ערימת שכבות: Frontend, Backend/API layer (השרת שלכם, שמחזיק את מפתח ה-API בצד השרת), שכבת אורקסטרציה, שכבת נתונים, Observability, ו-Deployment.

🔒 עיקרון אבטחה קריטי: לעולם לא קוראים ל-API של המודל ישירות מקוד frontend/דפדפן עם מפתח API מוטמע. הבקשה תמיד עוברת דרך ה-backend שלכם.

נקודה מעשית: מה שראינו כ"מלצר" בשיעור ה-API (מודול 1) הוא בעצם ה-backend שלכם - הוא זה שמדבר עם ה-API החיצוני, לא ה-frontend.

🔬 מעבדה: endpoint שרת שמשמש כ-proxy מאובטח
backend/routes/chat.js (pseudo)
app.post('/api/chat', authenticate, rateLimit, async (req, res) => {
  const { message } = req.body;
  const stream = await anthropic.messages.create({
    model: "claude-sonnet-5",
    messages: [{ role: "user", content: message }],
    stream: true,
    apiKey: process.env.ANTHROPIC_API_KEY
  });
  res.setHeader('Content-Type', 'text/event-stream');
  for await (const chunk of stream) res.write(chunk);
  logUsage(req.user.id, stream.usage);
  res.end();
});
💡 שימו לב לשלושה דברים: authenticate/rateLimit לפני שמגיעים למודל, מפתח ה-API רק ממשתנה סביבה בצד השרת, וlogUsage למעקב עלויות - כל אלה יתחברו הרבה יותר לעומק במודול 4.
